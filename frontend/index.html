How This Frontend Integrates With Django REST Framework

1. Modularity & Imports

Instead of copying API calling logic, we use standard ES6 JS modules. All requests go through js/api.js, importing methods using import { ... } from './api.js'.

2. HTTP Methods (REST)

We use specific HTTP actions mapping to database operations: GET for retrieval, POST for creation, PUT for replacements, and DELETE for removals.

3. Asynchronous Operations

We make use of modern JS fetch alongside async/await and try/catch blocks, allowing details to load dynamically without refreshing the page.
